18. Self-improvement loop
18.1 Purpose

Make the system get better after every campaign automatically.

This module connects everything:

campaign results → knowledge → ICP → signals → scoring → drafts

Without this, you just automate outbound.
With this, you build a system that learns and compounds.

18.2 Core idea

Every cycle should improve:

targeting
signal selection
messaging
prioritization

Instead of:

Campaign 1 → Campaign 2 (same quality)

You get:

Campaign 1 → learn → Campaign 2 better → learn → Campaign 3 even better
18.3 Loop stages

The loop runs in 5 steps:

1. Collect outcomes (campaign feedback)
2. Extract insights (knowledge base)
3. Generate improvement suggestions
4. Apply improvements (semi-auto)
5. Feed into next pipeline run

18.13 AI suggestion prompt
You analyze outbound campaign performance and suggest improvements.

Given:
- ICP definition
- campaign results
- reply classification
- signals
- segments
- knowledge items

Return JSON only:
{
  "icp_suggestions": [],
  "signal_suggestions": [],
  "persona_suggestions": [],
  "messaging_suggestions": [],
  "scoring_suggestions": [],
  "confidence_score": 0.0
}

Rules:
- Only suggest changes backed by data patterns.
- Prefer patterns seen across multiple leads.
- Do not overfit to small samples.
- Use cautious language when confidence is low.

18.18 Main flow
1. Load campaign data
2. Aggregate metrics
3. Analyze segments
4. Analyze signals
5. Analyze personas
6. Analyze messaging
7. Generate suggestions
8. Save suggestions
9. Optionally apply changes
10. Log improvement run
18.19 Pseudocode
class SelfImprovementModule(PipelineModule):
    name = "self_improvement"

    def run(self, context):
        campaigns = campaign_repo.get_recent(context.project_id)

        performance = performance_analyzer.compute(campaigns)

        segment_insights = segment_analyzer.analyze(performance)
        signal_insights = signal_analyzer.analyze(performance)
        persona_insights = persona_analyzer.analyze(performance)
        messaging_insights = messaging_analyzer.analyze(performance)

        suggestions = suggestion_engine.generate(
            segment_insights,
            signal_insights,
            persona_insights,
            messaging_insights
        )

        suggestion_repo.save(suggestions)

        if context.config.get("auto_apply"):
            applied = suggestion_applier.apply(suggestions)
        else:
            applied = 0

        improvement_repo.create(
            project_id=context.project_id,
            icp_id=context.icp_id,
            insights_generated=len(suggestions),
            applied_changes=applied
        )

        return ModuleResult(
            success=True,
            input_count=len(campaigns),
            output_count=len(suggestions),
            failed_count=0
        )

18.25 Definition of done

This module is complete when:

Campaign performance is analyzed
Signals are ranked by performance
Personas are ranked by performance
Messaging effectiveness is tracked
System generates structured improvement suggestions
Suggestions can be reviewed and applied
Future pipeline runs benefit from previous campaigns
The system improves over time instead of restarting from scratch
