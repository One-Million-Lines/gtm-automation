7. Contact Discovery Module
7.1 Purpose

Find the right people inside each qualified or enriched company.

The company may be a good fit, but outreach only works if we identify the correct person.

Output:

company → relevant contact(s) → lead candidate
7.2 Main goal

Given:

ICP personas + enriched companies

Find contacts such as:

Founder
CEO
CMO
Head of Growth
Head of CRM
Lifecycle Marketing Manager
Email Marketing Manager
Ecommerce Manager
Marketing Director
RevOps
Operations Manager
7.3 Input

Reads from:

companies
company_enrichment
icps
lead_candidates
suppression_list

Typical company input:

{
  "id": 12,
  "name": "Example Store",
  "domain": "examplestore.com",
  "linkedin_url": "https://linkedin.com/company/example-store",
  "industry": "DTC ecommerce",
  "status": "enriched"

{
  "target_personas": [
    "Founder",
    "Head of CRM",
    "Lifecycle Marketing Manager",
    "Ecommerce Manager"
  ]
}
7.4 Output

Writes to:

contacts
contact_sources
lead_candidates
module_logs

Each contact should include:

full name
job title
normalized role
company_id
LinkedIn URL
possible email if available
source URL
confidence score
7.5 What this module should not do

Do not:

deep validate emails
generate email drafts
score final lead quality
send outreach

It may find emails, but email validation belongs to the next module.

7.9 Main flow
1. Load enriched/qualified companies
2. Load ICP target personas
3. Run enabled contact providers
4. Normalize contact data
5. Match job title to persona
6. Check suppression
7. Deduplicate contacts
8. Save contacts
9. Save contact source
10. Create/update lead candidate with contact_id
11. Log result
7.10 Pseudocode
class ContactDiscoveryModule(PipelineModule):
    name = "contact_discovery"

    def run(self, context):
        icp = icp_repo.get(context.icp_id)

        companies = company_repo.get_for_contact_discovery(
            project_id=context.project_id,
            icp_id=context.icp_id,
            limit=context.config.get("limit", 50)
        )

        created = 0
        duplicates = 0
        failed = 0

        for company in companies:
            try:
                for provider in self.providers:
                    results = provider.discover(company, icp, context.config)

                    for raw_contact in results:
                        contact = normalize_contact(raw_contact, company)

                        if not contact.full_name:
                            failed += 1
                            continue

                        role_match = role_matcher.match(
                            job_title=contact.job_title,
                            personas=icp.target_personas
                        )

                        if not role_match.is_match:
                            continue

                        if suppression_service.is_contact_suppressed(contact):
                            continue

                        saved_contact, was_created = contact_repo.upsert(contact)

                        contact_source_repo.create(
                            contact_id=saved_contact.id,
                            source_type=provider.name,
                            source_url=contact.source_url,
                            raw_data=raw_contact,
                            confidence_score=role_match.confidence
                        )

                        lead_repo.create_if_missing(
                            project_id=context.project_id,
                            icp_id=context.icp_id,
                            company_id=company.id,
                            contact_id=saved_contact.id
                        )

                        if was_created:
                            created += 1
                        else:
                            duplicates += 1

            except Exception as e:
                failed += 1
                log_error(e, company.id)

        return ModuleResult(
            success=True,
            input_count=len(companies),
            output_count=created,
            failed_count=failed
        )

7.12 Role matching

This is critical.

The module should not save every random employee.
It should prioritize ICP personas.

Example mapping
Founder, Co-Founder, CEO → founder
CMO, VP Marketing, Marketing Director → marketing_lead
Head of Growth, Growth Lead → growth_lead
CRM Manager, Head of CRM → crm_lead
Lifecycle Marketing Manager → lifecycle_marketing
Email Marketing Manager → email_marketing
Ecommerce Manager, Head of Ecommerce → ecommerce_lead
Role match output
{
  "is_match": true,
  "normalized_role": "crm_lead",
  "persona_priority": 1,
  "confidence": 0.92,
  "reason": "Job title matches Head of CRM target persona"
}
7.13 Persona priority

ICP should define role priority.

Example:

[
  {
    "role": "Head of CRM",
    "priority": 1
  },
  {
    "role": "Lifecycle Marketing Manager",
    "priority": 2
  },
  {
    "role": "Founder",
    "priority": 3
  }
]

Use this later in scoring.

7.14 Deduplication rules

Primary keys:

email
linkedin_url
company_id + full_name

Rules:

if email exists → dedupe by email
else if LinkedIn exists → dedupe by LinkedIn
else → dedupe by company_id + normalized full_name

Do not create duplicates like:

Sarah Miller
Sarah A. Miller
S. Miller

Add fuzzy matching later.

7.15 Suppression checks

Suppress if:

email is in suppression list
LinkedIn URL is suppressed
company domain is suppressed
contact previously unsubscribed
contact bounced
contact manually rejected
7.16 Contact discovery status

New contact:

contacts.status = new

If matched to persona:

contacts.status = qualified

If wrong persona:

do not save, or save as rejected if useful

MVP recommendation:

only save matching personas
7.17 Contact confidence score

Calculate confidence from:

name found: +0.2
job title found: +0.2
role matches persona: +0.3
LinkedIn URL found: +0.2
email found: +0.1

Max:

1.0

Store in:

contact_sources.confidence_score

7.21 Contact ranking inside company

If multiple contacts are found, rank them.

Priority:

1. exact persona match
2. seniority
3. department relevance
4. has valid-looking email
5. has LinkedIn URL
6. source confidence

Example ranking:

Head of CRM > Marketing Manager > Founder > General Manager

For small companies:

Founder may be best

For larger companies:

functional owner is better
7.22 Recommended limits

Avoid too many contacts per company.

MVP:

max 3 contacts per company

Later:

max_contacts_per_company configurable per ICP

Example:

{
  "max_contacts_per_company": 3,
  "persona_priority": ["crm_lead", "growth_lead", "founder"]
}

7.26 Error handling

Item-level errors:

missing full name
missing job title
profile does not match company
provider returned duplicate
bad LinkedIn URL

Provider-level errors:

rate limited
invalid API key
scraper blocked
actor failed

Behavior:

log error
continue with next provider/company
do not stop full pipeline
