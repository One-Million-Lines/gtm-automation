11.1 Purpose

Prevent the system from creating or exporting:

duplicate companies
duplicate contacts
duplicate lead candidates
existing customers
competitors
unsubscribed contacts
bounced emails
already-contacted leads
manually rejected leads

This module protects outreach quality and avoids embarrassing mistakes.

11.2 Core idea

Separate two concepts:

Deduplication
This record already exists.

Example:

example.com already exists as a company
sarah@example.com already exists as a contact
Suppression
This record should not be used for outreach.

Example:

competitor domain
existing customer
unsubscribed email
bounced contact

11.10 Company merge behavior

If duplicate company is found:

do not create new company
attach new source to existing company
merge missing fields only
keep highest-confidence enrichment

Priority:

manual import > enrichment provider > scraped search result
11.11 Contact deduplication rules

Normalize email:

lowercase
trim whitespace

Normalize LinkedIn URL:

remove tracking params
remove trailing slash
normalize /in/ profile path

Normalize full name:

lowercase
remove punctuation
remove extra spaces

Dedupe priority:

1. email
2. linkedin_url
3. company_id + normalized full_name
11.12 Contact merge behavior

If duplicate contact is found:

do not create new contact
merge better email status
merge LinkedIn URL if missing
merge better job title if higher confidence
append contact source

Email priority:

valid > catch_all > risky > unknown > not_found > invalid
11.13 Lead candidate dedupe rules

Avoid:

same ICP + same company + same contact

If contact is null:

same ICP + same company + null contact

MVP implementation should handle this in code because SQLite uniqueness with null can be tricky.

11.14 Suppression rules

Suppress if:

company domain in suppression_list
company name in suppression_list
contact email in suppression_list
contact LinkedIn URL in suppression_list
contact previously unsubscribed
contact bounced
company is existing customer
company is competitor
lead already contacted
manual reject exists

Suppression always overrides score.

11.15 Suppression effects

If company is suppressed:

companies.status = suppressed
all related lead_candidates.lead_status = suppressed
all related lead_candidates.ready_for_outreach = 0

If contact is suppressed:

contacts.status = suppressed
related lead_candidates.lead_status = suppressed
ready_for_outreach = 0

If email is bounced or unsubscribed:

contacts.email_status = suppressed
contacts.status = suppressed
related lead_candidates.lead_status = suppressed
11.16 Campaign-based suppression

From campaign_events:

Unsubscribe
event_type = unsubscribed
→ add email to suppression_list
Bounce
event_type = bounced
→ add email to suppression_list
Already contacted
event_type = sent
→ lead_status = contacted

Depending on config:

do not contact same person again for X days
do not contact same company again for X days

11.23 Pseudocode
class DeduplicationSuppressionModule(PipelineModule):
    name = "deduplication_suppression"

    def run(self, context):
        company_merges = self.dedupe_companies(context)
        contact_merges = self.dedupe_contacts(context)
        lead_merges = self.dedupe_leads(context)

        suppressed_companies = self.apply_company_suppression(context)
        suppressed_contacts = self.apply_contact_suppression(context)
        suppressed_leads = self.apply_lead_suppression(context)

        return ModuleResult(
            success=True,
            input_count=company_merges + contact_merges + lead_merges,
            output_count=suppressed_companies + suppressed_contacts + suppressed_leads,
            failed_count=0,
            data={
                "company_merges": company_merges,
                "contact_merges": contact_merges,
                "lead_merges": lead_merges,
                "suppressed_companies": suppressed_companies,
                "suppressed_contacts": suppressed_contacts,
                "suppressed_leads": suppressed_leads
            }
        )
11.24 Company suppression pseudocode
def apply_company_suppression(context):
    companies = company_repo.get_active_companies(context.project_id)
    count = 0

    for company in companies:
        domain = normalize_domain(company.domain)

        if suppression_service.is_domain_suppressed(domain):
            company_repo.update_status(company.id, "suppressed")
            lead_repo.suppress_by_company(company.id)
            count += 1

    return count
11.25 Contact suppression pseudocode
def apply_contact_suppression(context):
    contacts = contact_repo.get_active_contacts(context.project_id)
    count = 0

    for contact in contacts:
        if contact.email and suppression_service.is_email_suppressed(contact.email):
            contact_repo.update_status(contact.id, "suppressed")
            lead_repo.suppress_by_contact(contact.id)
            count += 1

        if contact.linkedin_url and suppression_service.is_linkedin_suppressed(contact.linkedin_url):
            contact_repo.update_status(contact.id, "suppressed")
            lead_repo.suppress_by_contact(contact.id)
            count += 1

    return count
11.26 Campaign event suppression pseudocode
def apply_campaign_suppression():
    events = campaign_event_repo.get_unprocessed_suppression_events()

    for event in events:
        if event.event_type == "unsubscribed":
            suppression_service.suppress_email(
                email=event.contact.email,
                reason="unsubscribed",
                source="campaign_event"
            )

        if event.event_type == "bounced":
            suppression_service.suppress_email(
                email=event.contact.email,
                reason="bounced",
                source="campaign_event"
            )
