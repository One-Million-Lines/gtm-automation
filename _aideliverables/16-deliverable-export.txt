A row can be exported only if:

lead_candidates.ready_for_outreach = 1
lead_candidates.lead_status in ready, qualified, draft_approved
company is not suppressed
contact is not suppressed
email exists
email_status is valid or allowed
approved email draft exists
draft quality_status = passed
draft approved = 1
not already exported, unless force_export = true

16.8 Export batch table

Add export tracking.

CREATE TABLE export_batches (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    project_id INTEGER NOT NULL,
    icp_id INTEGER,

    export_type TEXT NOT NULL,
    destination TEXT,

    status TEXT DEFAULT 'created',

    file_path TEXT,
    total_rows INTEGER DEFAULT 0,

    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    completed_at TEXT,

    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (icp_id) REFERENCES icps(id)
);
16.9 Export row table

Track exactly what was exported.

CREATE TABLE export_rows (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    export_batch_id INTEGER NOT NULL,
    lead_candidate_id INTEGER NOT NULL,
    email_draft_id INTEGER,

    destination_row_id TEXT,

    status TEXT DEFAULT 'exported',

    exported_at TEXT DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (export_batch_id) REFERENCES export_batches(id),
    FOREIGN KEY (lead_candidate_id) REFERENCES lead_candidates(id),
    FOREIGN KEY (email_draft_id) REFERENCES email_drafts(id)
);

This prevents duplicates across exports.

16.10 Lead status after export

After successful export:

lead_candidates.lead_status = exported

Do not mark as contacted yet.

Contacted should happen only when:

campaign_events.event_type = sent
16.11 Duplicate export prevention

Before exporting, check:

export_rows
campaign_events
lead_candidates.lead_status

Skip if:

lead already exported
lead already contacted
contact already contacted
email already exists in active campaign

Unless:

force_export = true

16.14 Main flow
1. Load ready leads
2. Exclude already exported/contacted leads
3. Load approved draft
4. Load company/contact/research/signals
5. Build export rows
6. Create export batch
7. Generate destination output
8. Save export rows
9. Update lead statuses
10. Log result
16.15 Pseudocode
class ExportModule(PipelineModule):
    name = "export"

    def run(self, context):
        leads = lead_repo.get_ready_for_export(
            project_id=context.project_id,
            icp_id=context.icp_id,
            limit=context.config.get("limit", 500),
            include_exported=context.config.get("force_export", False)
        )

        rows = []

        for lead in leads:
            if export_repo.was_already_exported(lead.id) and not context.config.get("force_export"):
                continue

            draft = email_draft_repo.get_approved_passed_draft(lead.id)

            if not draft:
                continue

            row = export_builder.build_row(lead, draft)
            rows.append(row)

        batch = export_batch_repo.create(
            project_id=context.project_id,
            icp_id=context.icp_id,
            export_type=context.config.get("export_type", "csv"),
            destination=context.config.get("destination", "local_csv"),
            total_rows=len(rows)
        )

        file_path = csv_exporter.export(rows, batch.id)

        export_batch_repo.mark_completed(batch.id, file_path=file_path)

        for row in rows:
            export_row_repo.create(
                export_batch_id=batch.id,
                lead_candidate_id=row["lead_candidate_id"],
                email_draft_id=row["draft_id"]
            )

            lead_repo.update_status(row["lead_candidate_id"], "exported")

        return ModuleResult(
            success=True,
            input_count=len(leads),
            output_count=len(rows),
            failed_count=0,
            data={"file_path": file_path, "export_batch_id": batch.id}
        )
16.16 Export row builder

The builder should flatten all related data into one row.

class ExportBuilder:
    def build_row(self, lead, draft):
        company = company_repo.get(lead.company_id)
        contact = contact_repo.get(lead.contact_id)
        research = research_repo.get_latest(lead.id)
        signals = signal_repo.get_for_lead(lead.id)

        return {
            "lead_candidate_id": lead.id,
            "company_name": company.name,
            "domain": company.domain,
            "website_url": company.website_url,
            "industry": company.industry,
            "country": company.country,
            "contact_name": contact.full_name,
            "first_name": contact.first_name,
            "last_name": contact.last_name,
            "job_title": contact.job_title,
            "email": contact.email,
            "linkedin_url": contact.linkedin_url,
            "final_score": lead.final_score,
            "top_signal": get_top_signal(signals),
            "signal_sources": format_signal_sources(signals),
            "qualification_summary": lead.qualification_summary,
            "email_subject": draft.subject,
            "email_body": draft.body,
            "source_evidence": draft.source_evidence,
            "draft_id": draft.id
        }
