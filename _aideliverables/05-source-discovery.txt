5. Source Discovery Module
5.1 Purpose

Find companies that match an ICP and add them into the pipeline.

This module does not need to fully qualify the company.
Its job is to create a clean starting list of potential target companies.

Output:

Company records + source evidence + initial lead candidate
5.2 Main goal

Given an ICP, the module should answer:

Where can we find companies that look like this ICP?

Then it should collect:

company name
domain
website
source URL
reason discovered
raw source data
5.3 Input

From icps table:

industries
geographies
company size
competitors
buying signals
exclusion rules
target personas

Optional manual inputs:

seed companies
CSV upload
specific directories
specific competitors
specific search queries
specific Apify actors
5.4 Output

Writes to:

companies
company_sources
lead_candidates
module_logs

Example output:

{
  "name": "Example Store",
  "domain": "examplestore.com",
  "website_url": "https://examplestore.com",
  "source_type": "google_search",
  "source_url": "https://www.google.com/search?q=site...",
  "discovery_reason": "Found while searching for Shopify fashion stores using Klaviyo",
  "icp_id": 1
}
5.5 What this module should not do

Do not overload this module.

It should not:

find contacts
write email drafts
fully qualify leads
score leads deeply
validate emails

It can add a light discovery confidence score, but deeper qualification happens later.

5.9 Main module flow
1. Load ICP
2. Load active discovery providers
3. Build discovery queries
4. Run providers
5. Normalize company data
6. Extract domain
7. Check suppression
8. Deduplicate
9. Save company
10. Save company source
11. Create lead candidate
12. Log result
5.10 Pseudocode
class SourceDiscoveryModule(PipelineModule):
    name = "source_discovery"

    def run(self, context):
        icp = icp_repo.get(context.icp_id)
        providers = self.get_enabled_providers(context.config)

        total_found = 0
        total_created = 0
        total_duplicates = 0
        total_failed = 0

        for provider in providers:
            results = provider.discover(icp, context.config)

            for item in results:
                try:
                    normalized = normalize_company(item)

                    if not normalized.domain:
                        total_failed += 1
                        continue

                    if suppression_service.is_domain_suppressed(normalized.domain):
                        continue

                    company, created = company_repo.upsert_by_domain(normalized)

                    company_source_repo.create(
                        company_id=company.id,
                        source_type=provider.name,
                        source_url=normalized.source_url,
                        raw_data=item
                    )

                    lead_repo.create_if_missing(
                        project_id=context.project_id,
                        icp_id=context.icp_id,
                        company_id=company.id
                    )

                    total_found += 1

                    if created:
                        total_created += 1
                    else:
                        total_duplicates += 1

                except Exception as e:
                    total_failed += 1
                    log_error(e, item)

        return ModuleResult(
            success=True,
            input_count=total_found,
            output_count=total_created,
            failed_count=total_failed
        )

5.13 Normalization rules

Every provider returns messy data.

Normalize to this structure:

{
    "name": str,
    "domain": str,
    "website_url": str,
    "linkedin_url": str | None,
    "country": str | None,
    "city": str | None,
    "industry": str | None,
    "description": str | None,
    "source_url": str | None,
    "source_type": str,
    "raw_data": dict
}
5.14 Domain extraction rules

This is critical for deduplication.

Examples:

https://www.example.com/pages/about → example.com
http://example.com → example.com
www.example.co.uk → example.co.uk
shop.example.com → example.com, unless configured otherwise

Use a domain parser library if possible.

Recommended Python packages:

tldextract
urllib.parse
5.15 Deduplication rules

Primary dedupe key:

domain

Secondary fallback:

normalized company name + country

Company name normalization:

remove Inc, LLC, GmbH, AG, Ltd
lowercase
remove punctuation
trim whitespace

Examples:

Example GmbH
Example Ltd.
EXAMPLE
→ example
5.16 Suppression checks

Before saving, check:

suppression_list

Suppress if:

domain is competitor
domain is existing customer
domain is blacklisted
domain was unsubscribed
domain previously bounced too much
company manually rejected
5.17 Company discovery status

When saved:

companies.status = discovered
lead_candidates.lead_status = new

If already exists:

do not overwrite important fields unless new data is better
5.18 Discovery confidence score

Each source should assign an initial confidence.

Example:

manual CSV: 1.0
BuiltWith competitor list: 0.9
Google result: 0.6
Job post mention: 0.7
LinkedIn search: 0.8

Store in:

company_sources.confidence_score

5.23 Data quality rules

Reject or mark failed if:

no company name
no domain and no website
invalid domain
domain is social-only like linkedin.com/company/...
domain is marketplace profile without real website

Needs review if:

company name exists but domain unclear
source confidence < 0.5
duplicate name but different domain

5.26 Error handling

Item-level errors:

missing domain
invalid URL
parse failure
suppressed domain
duplicate company

Provider-level errors:

API unavailable
rate limited
bad credentials
actor failed
invalid response format

Provider failure should not kill the full pipeline unless configured.
