4. ICP Definition & Refinement Module

This is one of the most important parts.
If ICP is weak → everything downstream is garbage.

Goal:

Define WHO you target + WHY they should care + HOW you approach them
4.1 Purpose

This module allows you to:

define a structured ICP
refine it over time
connect ICP to signals + scoring + messaging
feed everything downstream (discovery, scoring, drafts)
4.2 Output of this module

Final output stored in DB (icps):

- target companies (filters)
- target personas (roles)
- competitors (for signals)
- pain points (for messaging)
- buying signals (for triggers)
- exclusions (for filtering)
- outreach angle (for email drafts)

Everything else in the system depends on this.

4.10 Refinement logic
Step 1: extract patterns

From replies:

positive replies → what worked
negative replies → objections
no replies → weak messaging
Step 2: classify insights

Example:

"we already use Klaviyo"
→ competitor signal

"too expensive"
→ pricing objection

"not relevant"
→ bad ICP
Step 3: store in knowledge_items
{
  "type": "objection",
  "content": "too expensive",
  "icp_id": 1,
  "importance": 0.8
}
Step 4: update ICP fields

Examples:

add new competitor
refine pain points
add/remove signals
change outreach angle

4.18 Common mistakes (avoid these)
1. Too broad ICP

Bad:

"all SaaS companies"

Good:

"SaaS companies using HubSpot with 10–200 employees hiring marketing roles"
2. No signals

Bad:

"target ecommerce companies"

Good:

"ecommerce companies using Klaviyo or hiring CRM roles"
3. Generic pain points

Bad:

"improve marketing"

Good:

"manual campaign setup taking too long"
4. No exclusions

You will spam:

agencies
enterprises
existing customers
competitors
